- [Meta OAuth callback URL](meta-oauth-callback.md) — must use Replit dev domain without port; port-suffixed URLs break Meta's redirect_uri validation.
- [Production hardening patterns](production-hardening.md) — null-safe initials, ErrorBoundary, DEFAULT_STATUS_CONFIG fallback, analytics array defaults — all recurring safety patterns in this CRM.
- [API server port conflict](api-server-port.md) — port 8080 zombie processes cause EADDRINUSE on restart; kill PIDs from ps aux before restarting the artifact workflow.
- [Meta Lead Ads sync architecture](meta-lead-sync.md) — pluggable sync service in services/metaLeadSync.ts; scheduled every 15min; dedupes by externalId column; add future platforms alongside runMetaLeadSync().
- [Settings theme override bug](settings-theme.md) — never call setTheme() in a useEffect that watches settings data; next-themes owns the theme via localStorage; only call setTheme() on explicit user clicks.
- [Leads schema user isolation](leads-isolation.md) — leads table had no created_by_id column (added June 2026); GET routes use OR created_by_id IS NULL to preserve legacy data; analytics raw SQL must match.
- [Data isolation pattern](data-isolation.md) — leads/deals/analytics filter by userId; use OR col IS NULL for nullable FK columns to handle pre-isolation data; properties uses listed_by_id, deals uses created_by_id.
- [Post-login redirect race condition](post-login-redirect.md) — NEVER call setLocation("/dashboard") after signInWithPassword; let PublicOnlyRoute detect the new session and redirect; manual navigation fires before React re-renders the auth context.
- [OnboardingGuard isError handling](onboarding-guard-iserror.md) — Do NOT redirect on isError; only redirect when profile.onboarded===false. New users handled by sign-up page redirect. Redirecting on errors bounces existing users on transient API failures.
- [Google OAuth redirectTo BASE_URL](google-oauth-redirect.md) — redirectTo must include import.meta.env.BASE_URL prefix: `${origin}${BASE_URL.replace(/\/$/,"")}/dashboard`. Without it, Google OAuth lands on root `/` not the app's sub-path.
- [Onboarding save must check response](onboarding-save-check.md) — fetch() does NOT throw on HTTP errors; PUT /api/users/me must check res.ok and throw if failed or onboarded:true is silently skipped causing an infinite redirect loop.
